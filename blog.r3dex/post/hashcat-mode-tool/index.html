Hashcat Made Finding Modes Harder, So I Built a Tool to Fix It

Introduction

In the fast-paced world of penetration testing and Capture The Flag (CTF) competitions, time is your most valuable asset. When you extract a password hash during an engagement, the immediate next step is attempting to crack it. While Hashcat remains the industry standard for password recovery, recent changes to its documentation and reference pages have made finding the exact hash mode (-m flag) more cumbersome than it used to be.

Identifying the correct hash algorithm is step zero in password cracking. If you guess the wrong mode, Hashcat will either fail to execute or waste precious GPU cycles. To bridge this gap and streamline the workflow, I developed a dedicated Hash Identification Tool. Designed to instantly map unknown hashes to their corresponding Hashcat modes, this tool ensures you can get straight to cracking without scouring wikis.

Features

To accommodate different workflows, the project was built with two distinct versions:

  • Web-Based Interface: A clean, accessible GUI designed for beginners or professionals who need a quick, on-the-fly lookup without installing dependencies.
  • CLI (Command-Line) Version: A robust, terminal-native script built for advanced users, penetration testers, and seamless integration into automated workflows.
  • Extensive Database: Supports a massive array of common and obscure hash formats recognized by modern Hashcat.
  • Ranking System: Intelligently handles ambiguous hashes by returning the most probable matches first.

How It Works

Under the hood, the tool acts as a specialized pattern recognition engine. When a hash is submitted, the engine evaluates several cryptographic characteristics:

  • String Length: The tool measures the exact length of the input. For example, a 32-character string immediately filters the possibilities down to MD5, NTLM, MD4, etc.
  • Character Sets: It checks if the hash consists solely of hexadecimal characters, includes base64 encoding, or features special alphanumeric combinations.
  • Prefix Signatures: Many modern hashing algorithms use structural identifiers. The tool actively looks for known prefixes, such as $1$ (MD5crypt), $2a/b/y$ (bcrypt), or $6$ (SHA512crypt), to guarantee a precise match.

Installation (CLI Version)

Getting the CLI version up and running on your local machine or attack box is straightforward. Simply clone the repository and run the tool:

# Clone the repository
git clone https://github.com/karthikparambil/hasmode.git

# Navigate into the directory
cd hasmode

# (Optional) Install any required dependencies if specified in the repo
pip install -r requirements.txt

# Make the script executable
chmod +x hasmode.py

Usage

The CLI tool is designed to be highly intuitive. You can pass a hash directly as an argument.

Command:

python3 hasmode.py -H "5d41402abc4b2a76b9719d911017c592"

Expected Output:

[*] Analyzing Hash...
[+] Hash length: 32
[+] Character set: Hexadecimal
--------------------------------------------------
[!] Possible Matches Found:
 1. MD5 -> Hashcat Mode: 0
 2. NTLM -> Hashcat Mode: 1000
 3. MD4 -> Hashcat Mode: 900

The Web Version

Not everyone wants to drop into a terminal for a single hash lookup. The Web Version provides a frictionless alternative. Featuring a minimalist UI, users can simply paste their hash into the search bar and hit "Identify."

The primary benefit of the web interface is accessibility—it requires no installation, is OS-agnostic, and is perfect for quick reference during a CTF when you already have multiple terminal windows open.

Example

Let’s look at how the tool handles a more complex, salted hash.

Input: $2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy

Output:

[+] Hash Type: bcrypt
[+] Hashcat Mode: 3200
[+] Description: Blowfish (OpenBSD)

Instead of guessing, you instantly know to run: hashcat -m 3200 hash.txt wordlist.txt.

Challenges Faced

Building a reliable hash identifier comes with distinct technical hurdles. The biggest challenge was dealing with hash collisions and similarities. Many algorithms output standard 32, 40, or 64-character hexadecimal strings (e.g., MD5 vs. NTLM). Because it is mathematically impossible to differentiate them with 100% certainty just by looking at the string, I had to implement a logical fallback. Instead of forcing a single, potentially incorrect answer, the tool outputs a ranked list of "Probable Matches" based on commonality, allowing the user to test the most likely options first.

Handling false positives with complexly formatted enterprise hashes (like custom salted hashes or obscure database formats) also required careful tuning of the regular expressions used for pattern matching.
